What is the Document Object Model (DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document as a tree of objects, allowing programming languages like JavaScript to interact with HTML and XML documents. Each element, attribute, and text in the document is represented as a node in this tree, enabling developers to read, manipulate, and modify the content and structure of web pages dynamically.

How do you select an element from the DOM?

You can select elements from the DOM using several methods in JavaScript:

  1. 1. getElementById: Selects a single element by its ID.
  2. 2. getElementsByClassName: Selects elements with a specific class name (returns a live HTMLCollection).
  3. 3. getElementsByTagName: Selects elements by their tag name (also returns a live HTMLCollection).
  4. 4. querySelector: Selects the first element that matches a CSS selector.
  5. 5. querySelectorAll: Selects all elements that match a CSS selector (returns a static NodeList).

What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique that involves setting a single event listener on a parent element instead of on multiple child elements. This takes advantage of event bubbling, where events propagate from the target element up to its parent elements. Benefits of event delegation include: Performance: Reduces the number of event listeners, improving performance, especially with many child elements. Dynamic elements: Automatically handles events for dynamically added child elements without needing to reattach listeners. Cleaner code: Simplifies event handling logic by centralizing it in one place.
